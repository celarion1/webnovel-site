# 내 웹소설 서재

여러 편의 웹소설을 "제목 → 회차 목록 → 본문" 구조로 올려서 링크 하나로 공개할 수 있는 사이트입니다.
Next.js + Vercel 조합이며, 화면은 위쪽 하늘/구름 헤더, 왼쪽 목록/관리 패널(사이드바), 오른쪽 읽기 영역 3단 구성입니다.

## 이번 업데이트: 배포된 사이트에서 바로 등록·수정·삭제 (관리자 로그인)

이제 배포된 사이트(vercel.app 링크) 자체에서 관리자 비밀번호로 로그인하면 등록·수정·삭제가 가능하고, 로그인하지 않은 방문자는 읽기만 가능합니다.

- **관리자(나)**: 사이트에 접속해 왼쪽 패널의 "관리자 로그인"에 비밀번호를 입력 → 그 브라우저는 계속 관리자 상태 유지(최대 1년) → 등록/수정/삭제/순서변경 가능
- **다른 사람**: 같은 링크로 접속해도 로그인하지 않으면 소설 목록과 회차를 읽기만 할 수 있음 (등록/수정/삭제 버튼 자체가 보이지 않음)

즉, 링크는 하나지만 로그인 여부에 따라 같은 브라우저는 관리자로, 다른 사람의 브라우저는 읽기 전용으로 동작합니다. 로그아웃하면 그 브라우저도 다시 읽기 전용이 됩니다.

이 기능이 배포 사이트에서 실제로 저장되려면(Vercel은 파일을 저장할 수 없는 환경이라) 아주 작은 무료 데이터베이스(Redis)를 하나 연결해야 합니다. 아래 순서대로 진행하면 됩니다.

## 1. 로컬에서 실행하기 (데이터베이스 없이도 동작)

```bash
npm install
npm run dev
```

브라우저에서 http://localhost:3000 접속. 로컬에서는 데이터베이스 없이 기존처럼 `content/` 폴더 파일을 그대로 사용합니다. 다만 관리자 로그인은 로컬에서도 비밀번호가 필요합니다 — 아래 2번에서 설정하세요.

## 2. 관리자 비밀번호 설정

### 로컬용

프로젝트 루트에 `.env.local` 파일을 만들고(`.env.local.example` 참고):

```
ADMIN_PASSWORD=원하는_비밀번호
```

저장 후 `npm run dev`를 다시 시작하면, 로컬 사이트에서도 이 비밀번호로 로그인해야 등록/수정/삭제 버튼이 나타납니다.

### 배포용 (Vercel)

Vercel 프로젝트 → **Settings → Environment Variables** 에서 `ADMIN_PASSWORD` 이름으로 원하는 비밀번호를 추가하고 저장합니다. (로컬과 같은 값을 써도 되고 달라도 됩니다.)

## 3. 실시간 저장을 위한 데이터베이스(Redis) 연결

배포된 사이트에서 로그인 후 등록한 내용이 실제로 저장되려면 Redis(Upstash) 연동이 필요합니다. 무료 등급으로 충분합니다.

1. Vercel 프로젝트 대시보드 → **Storage** 탭 → **Create Database**(또는 Marketplace Database) → **Upstash** (Redis/KV) 선택 → 안내에 따라 생성 후 프로젝트에 연결(Connect)합니다.
2. 연결이 끝나면 Vercel이 프로젝트의 Environment Variables에 Redis 접속 정보를 자동으로 추가해줍니다. 이름이 `KV_REST_API_URL` / `KV_REST_API_TOKEN` 이거나 `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN` 일 수 있는데, 둘 다 이 사이트 코드가 자동으로 인식합니다.
3. 로컬에서도 같은 값을 `.env.local`에 추가해두면(4번 마이그레이션 스크립트 실행을 위해 필요) 됩니다. Vercel 대시보드의 Environment Variables 화면에서 값을 복사해 붙여넣으면 됩니다.

이 단계까지 마치고 배포하면, Vercel에 올라간 사이트는 파일이 아니라 이 Redis를 실제 저장소로 사용합니다(코드가 자동으로 감지해서 전환합니다).

## 4. 기존에 로컬에 등록해둔 소설을 Redis로 옮기기 (한 번만)

`.env.local`에 `ADMIN_PASSWORD`와 Redis 접속 정보(3번)를 모두 넣은 상태에서:

```bash
npm run migrate-to-redis
```

`content/` 폴더에 있는 소설·회차를 통째로 Redis에 복사합니다(이미 있는 소설은 건너뜁니다). 실행 후 배포 사이트에 관리자로 로그인하면 그대로 보입니다.

## 5. GitHub에 올리기

이미 GitHub 저장소가 있다면 변경된 파일만 커밋해서 올리면 됩니다:

```bash
git add .
git commit -m "관리자 로그인 및 실시간 등록/수정/삭제 기능 추가"
git push
```

처음 올리는 경우:

```bash
git init
git add .
git commit -m "첫 커밋"
git branch -M main
git remote add origin https://github.com/<내계정>/<저장소이름>.git
git push -u origin main
```

**주의:** `.env.local` 파일은 비밀번호와 DB 접속 정보를 담고 있으므로 `.gitignore`에 의해 자동으로 git에서 제외됩니다. 절대 직접 커밋하지 마세요. 실제 값은 Vercel 대시보드의 Environment Variables에만 입력합니다.

## 6. Vercel로 배포하기

1. https://vercel.com 에 GitHub 계정으로 로그인 후 "Add New... → Project" → 방금 올린 저장소 선택(Import).
2. 2번에서 설정한 `ADMIN_PASSWORD`와 3번에서 연결한 Redis가 이미 Environment Variables에 들어있는지 확인합니다.
3. "Deploy" 클릭. 완료되면 `https://<프로젝트이름>.vercel.app` 링크가 생성됩니다.

이 링크가 곧 "읽기 전용 공유 링크"입니다. 이 링크로 들어가서 왼쪽 패널의 "관리자 로그인"에 비밀번호를 입력하면 그 브라우저에서만 관리자 기능이 열립니다. 다른 사람에게는 같은 링크를 그대로 공유하면 되고, 로그인하지 않는 한 읽기만 가능합니다.

## 7. 이후 사용법

- **관리자로서 글 올리기**: 배포된 링크 → 관리자 로그인 → 왼쪽 패널에서 등록/수정/삭제/순서변경. 저장 즉시 모든 방문자에게 반영됩니다(별도 git push 불필요).
- **로컬 개발**: `npm run dev`는 여전히 사용 가능하며, Redis 접속 정보를 `.env.local`에 넣지 않으면 로컬은 파일 기반으로 동작합니다(빠른 테스트용).
- 코드 자체를 수정한 경우에만 git push가 필요합니다. 글 내용(소설/회차) 등록·수정·삭제는 이제 배포 사이트에서 바로 이루어집니다.

## 7-1. 방문자수·조회수 통계

별도 설정 없이 자동으로 동작합니다(3번에서 연결한 Redis를 그대로 사용).

- **집계 항목**: 일자별 전체 조회수, 일자별 순 방문자수(같은 브라우저는 하루에 1명으로 집계), 회차별 누적 조회수.
- **확인 방법**: 배포된 사이트 → 관리자 로그인 → 왼쪽 패널의 "📊 통계 보기" 클릭. 최근 30일 일별 표와 회차별 조회수 TOP 10이 표시됩니다.
- 방문자 식별은 브라우저 쿠키(1년 유지) 기준이며, 로그인 없이도 모든 방문자에게 자동으로 부여됩니다. 관리자 본인의 방문·클릭도 함께 집계되니 참고하세요.
- 로컬 파일 모드(Redis 미연결)에서는 `content/analytics.json`에 기록되며 git에는 올라가지 않습니다(`.gitignore` 처리됨).

## 8. 데이터 구조 (참고용)

- **Redis 연결 시**: `webnovel:data` 라는 키 하나에 모든 소설/회차가 JSON으로 저장됩니다.
- **로컬 파일 모드**: 기존과 동일하게 `content/<slug>/meta.json` + `content/<slug>/episodes/<id>.txt` 구조를 사용합니다.

커맨드라인으로도 등록 가능합니다(로컬 파일 모드에서만 동작): `npm run add-novel -- "제목" "소개"`, `npm run add-episode -- <slug> "회차 제목" ./원고.txt`.

## 9. 참고

- 회차 파일/텍스트 인코딩은 UTF-8을 권장합니다.
- 소설/회차 삭제는 되돌릴 수 없습니다. 중요한 원고는 별도로 백업해두세요.
- 관리자 비밀번호는 브라우저 쿠키에 저장되어 로그인 상태를 유지합니다. 공용 컴퓨터에서 로그인했다면 반드시 로그아웃하세요.
